Gradio GUI for live predictions, organized into the project layout below.

## Dataset

`Data/raw/emotion_data.csv` ‚Äî 480 short text samples labelled across 6 balanced emotion
classes (80 each): `joy`, `sadness`, `anger`, `fear`, `surprise`, `neutral`.

```bash
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## How to Run

```bash
jupyter notebook Notebook/emotion_prediction_system_gui.ipynb
```

Run the cells in order:
1. **Imports ‚Üí EDA ‚Üí preprocessing ‚Üí train/test split ‚Üí TF-IDF features**
2. **Model selection** ‚Äî compares Logistic Regression, Linear SVC, Multinomial NB, and
   Random Forest on test accuracy and picks the best.
3. **Evaluation** ‚Äî classification report and confusion matrix for the best model.
4. **Save model** ‚Äî writes `best_emotion_model.pkl` and `tfidf_vectorizer.pkl` into `Model/`.
5. **Inference + Gradio GUI** ‚Äî loads the saved model/vectorizer from `Model/` and launches an
   interactive Gradio interface for live text-to-emotion predictions.

> File paths inside the notebook (`../Data/raw/emotion_data.csv`, `../Model/*.pkl`) are
> already set up for this folder layout ‚Äî the notebook expects to be run from inside
> `Notebook/`, which is the default when you open it via Jupyter as shown above.

The notebook's final section (pushing the trained model to GitHub via a Colab secret token)
is specific to running in Google Colab and can be skipped when running locally.
